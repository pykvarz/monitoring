# Документация по безопасности и соответствию требованиям

Этот документ предоставляет технические детали для отделов информационной безопасности (ИБ) касательно приложения "Network Monitor".

## Поведение приложения
**Network Monitor** — это инструмент для пассивного и активного сетевого мониторинга.
- **Активность**: Приложение отправляет стандартные **ICMP Echo Requests** (Ping) на указанные IP-адреса для проверки их доступности.
- **Протокол**: ICMP (Internet Control Message Protocol).
- **Частота**: Настраиваемый интервал опроса (по умолчанию: 10 секунд).
- **Область действия**: Приложение взаимодействует **только** с конкретными IP-адресами, указанными пользователем в конфигурации. Оно не выполняет сканирование сети, сканирование портов или поиск уязвимостей, если это явно не изменено в коде.

## Конфиденциальность данных и хранение
- **Локальное хранение**: Все данные (список узлов, история, логи) хранятся **локально** на компьютере пользователя.
  - База данных: SQLite (`hosts.db`) или JSON в папке приложения.
  - Логи: Локальный текстовый файл (`debug.log`).
- **Телеметрия**: Приложение **НЕ** содержит внешней телеметрии, трекеров или аналитики. Никакие данные не отправляются на сторонние серверы.

## Соответствие требованиям ИТ-безопасности
### 1. Цифровая подпись
Исполняемый файл (`NetworkMonitor.exe`) **не подписан** цифровой подписью.
- **Причина**: Это внутренний/open-source инструмент, собранный локально.
- **Влияние**: Windows SmartScreen или антивирусное ПО могут помечать файл как "Неизвестный издатель" или "Редкая загрузка".
- **Решение**: ИТ-администраторы могут проверить исходный код (Python) и собрать исполняемый файл самостоятельно для обеспечения доверия.

### 2. Сетевые разрешения и Права Администратора
- **Права Администратора**: Не обязательны.
  - *С правами администратора*: Приложение использует "сырые сокеты" (raw sockets) через библиотеку `ping3`. Это работает быстрее и создает меньше нагрузки на систему.
  - *Без прав администратора*: Приложение автоматически переключается на использование системной утилиты `ping.exe`. Это полностью поддерживаемый режим.
- Правила фаервола должны разрешать **исходящий ICMP** трафик.

### 3. Прозрачность исходного кода
Приложение создано с использованием стандартных библиотек Python:
- `PyQt5` (GUI)
- `sqlite3` (База данных)
- стандартные библиотеки (`subprocess`, `threading` и др.)

Полный исходный код доступен для аудита.

## Ложные срабатывания антивируса
Антивирусное ПО может иногда помечать приложения, собранные с помощью `PyInstaller`, как ложные срабатывания (false positives). Это известная особенность эвристического анализа загрузчиков Python.
- **Проверка**: Вы можете загрузить `.exe` файл на [VirusTotal](https://www.virustotal.com/) для проверки, или запускать приложение напрямую из исходного кода Python.
